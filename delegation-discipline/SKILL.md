## 2. HOW TO DELEGATE

### THE 3 RULES OF GOOD DELEGATION

#### Rule 1: Write Clear, Single-Purpose Briefs
**BAD:**
> "Do some research on coconut oil products and find me good ones to sell"

**GOOD:**
> "Search Taobao for organic virgin coconut oil 500ml-1L, filter by rating ≥4.5, sales ≥500. Return top 5 products with price, shop name, and URL. Focus on Malaysia logistics."

#### Rule 2: Batch Related Tasks
**BAD:**
> "What's the weather?"
> "What's the time?"
> "Check my calendar"
> "Any new emails?"

**GOOD:**
> "Quick status check: weather, time, calendar events in next 24h, unread emails. One-liner summary."

#### Rule 3: Trust the Brief, Don't Micromanage
**BAD:**
> "Find products" → "Not this one" → "Try something else" → "Change filter" → "Adjust search"

**GOOD:**
> "Find organic coconut oil with these specs: price $40-60, rating ≥4.5, sales ≥1000. Return top 5."

### THE DELEGATION CHECKLIST

Before delegating, ask:
1. ✅ Is this >1 tool calls? → Delegate
2. ✅ Is this someone else's core skill? → Delegate
3. ✅ Can this be batched? → Batch and delegate
4. ✅ Do I have the full brief written? → Yes, send
5. ✅ Am I micromanaging? → Stop, trust the brief

## 4. TOKEN DISCIPLINE

### THE 3 PRINCIPLES

#### Principle 1: Never Burn Tokens on Lookups

**BAD (burning tokens):**
> "What's the weather in KL? What time is it? Check my calendar for tomorrow? Any new emails?"

**GOOD (delegating):**
> "Quick status: weather, time, calendar, emails. One-liner."

**Rule:** If you need >3 lookups, delegate to Myrmidons.

#### Principle 2: Write Files, Don't Keep in Head

**BAD:**
- Remembering 20 product URLs in memory
- Reciting complex configurations
- Keeping multiple task states

**GOOD:**
- Save to `products.md`
- Document in `config.json`
- Write to `tasks.json`

**Rule:** If you need to remember >5 items, write to file.

#### Principle 3: Compact Proactively

**When to compact:**
- After 3+ tasks completed
- After handling >10 messages
- Before starting complex multi-step work
- When memory feels "full"

**Rule:** Don't wait for warnings — be proactive.

### TOKEN SAVINGS EXAMPLE

**Scenario:** Research 10 products

**Bad (no delegation):**
- 10 × (search + extract) × 1000 tokens = 10,000 tokens

**Good (delegate to Artemis):**
- Brief + 1 tool call = ~200 tokens
- Artemis research = 500 tokens
- Total: ~700 tokens

**Savings: 93% fewer tokens**

## DELEGATION PATTERNS

### Pattern 1: The "Swarm" Delegation

**Use when:** Dozens of similar tasks

```
Zenni → Myrmidons:
"Process these 50 WhatsApp messages:
- Extract product mentions
- Count sentiment
- Categorize by topic
Return summary table."
```

### Pattern 2: The "Assembly Line"

**Use when:** Multi-step process with clear handoffs

```
Zenni → Artemis → Apollo → Artee → Hermes:
"Research organic skincare products (Artemis)
→ Write copy for ad campaign (Apollo)
→ Design visual assets (Artee)
→ Source products and pricing (Hermes)"
```

### Pattern 3: The "Quick Status"

**Use when:** Multiple quick checks needed

```
Zenni → Myrmidons:
"Quick status: weather, time, unread emails, calendar events.
One-liner summary."
```

## COMMON MISTAKES

### Mistake 1: Doing Your Own Lookup
**Symptom:** "Let me check that for you" when asking humans

**Fix:** "Myrmidons, do quick lookup on [topic]"

### Mistake 2: Micromanaging Delegate
**Symptom:** "Not quite, try this..." after delegation

**Fix:** Write clearer brief, trust the agent

### Mistake 3: Not Batching
**Symptom:** Multiple separate tool calls for related tasks

**Fix:** "Handle these 5 tasks in batch: [list]"

### Mistake 4: Keeping in Head
**Symptom:** Reciting info instead of writing

**Fix:** "Write to memory/file" before continuing
