## Universal Filing System

Master doc: `_WORK/_shared/FILING-SYSTEM.md`

### Core Rules
1. Every file has ONE home — no duplicates across brands
2. Universal research (2+ brands) → `_shared/03_research/[topic]/`
3. Brand-specific → `_WORK/[brand]/[numbered-folder]/`
4. Reference images MUST be classified before saving: FORMAT / AESTHETIC / CONTENT / FOOD
5. Only LATEST script version in 05_scripts/ — old versions archived to 07_working/
6. Data files (.json, .csv) belong in 05_scripts/data/ — NOT in 02_strategy/
7. Max 1 file at brand root (session handoff doc)
8. Rejected outputs: log reason → extract rule → DELETE file
9. No empty placeholder folders — create only when you have files to put in them
10. No version suffixes in folder names (curated/ not curated-v2/)

### Decision Tree (for ANY new file)
- Universal? → `_shared/`
- Strategy? → `02_strategy/` (archive old version if replacing)
- Reference image? → `04_references/[FORMAT|AESTHETIC|CONTENT|FOOD]/`
- Production output? → `06_exports/` (if rejected: log + delete)
- Script? → `05_scripts/` (archive old version)
- Raw asset? → `01_assets/`
- Research? → Brand-specific `03_research/` or universal `_shared/03_research/`
- Temporary? → `07_working/` (clean up within 30 days)

**Why:** Consistency across brands lets any agent/collaborator find files in <5 seconds. No "which folder was that in?" ever again.

**How to apply:** Read `_WORK/_shared/FILING-SYSTEM.md` at start of any session that creates files. Follow the decision tree. Run hygiene check after every batch.
