# NANO Illustration Quality — Obsessed With Perfection

**Trigger**: User called out lazy illustrations when I focused on typography fixes and let illustration prompts get thin/generic. "You got a bit lazy on the illustrations. You are obsessed with perfection only. No cutting corners. Like this you're wasting tokens."

## The Rule

Every NANO illustration prompt must be **richly, vividly, specifically art-directed**. No bare-bones descriptions. Every character, object, and scene element needs:

### Characters MUST have:
- **Specific expression**: "wide sparkling eyes, huge proud grin, cheeks slightly flushed"
- **Specific pose/action**: "tongue sticking out in concentration", "one arm reaching up toward the words"
- **Specific accessories**: "tiny brown leather explorer hat tilted slightly to one side"
- **Personality**: each character doing something DIFFERENT and telling a STORY

### Scenes MUST have:
- **Rich object detail**: not "coins and eggs" but "stacks of colorful play money fanned out, piles of coins in yellow and copper, a tiny toy cash register in yellow"
- **Storytelling props**: "craft supplies (scissors, glue stick, glitter tube)", "scattered game cards with simple illustrations on them"
- **Texture/quality calls**: "risograph printing texture quality — tactile, handmade, editorial"
- **Reward for looking closely**: "The illustration is RICH and detailed — reward the viewer for looking closely"

### Objects MUST have:
- **Visual variety**: "one yellow with polka dots, one coral with stripes, one blue with zigzags, one lavender with tiny stars"
- **State variety**: "some cracked, some whole, some decorated"
- **Brand color mapping**: every object maps to a specific brand color

## Anti-pattern (NEVER do this)
```
"Scatter 4-5 objects on the ground around the character. Add 3 tiny floating sparkles."
```
This is LAZY. It produces generic, forgettable illustration.

## Correct pattern
```
"5-6 [objects] scattered on the ground — two are [specific state] with [specific contents]
spilling out ([item] in [brand color 1], [item] in [brand color 2]). One [object] has a
tiny [surprise element] peeking out. A small [container] sits beside the character with
2 [objects in different state] inside."
```
Every object has: specific color, specific state, specific relationship to other objects.

## Why this matters
- Thin prompts = generic AI output = wasted API tokens
- Rich prompts = distinctive, charming, premium output = worth the cost
- The user is paying per generation — every token must count toward quality
