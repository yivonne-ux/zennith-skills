Never build design output from scratch. Always use design intelligence pipeline:
1. Scrape world-class references first
2. Analyze reference design DNA (layout, typography, color, spacing, mood)
3. Input brand data + reference analysis into AI generation (NANO)
4. Design = AI creative work, NOT programmatic PIL text overlay

**Why:** PIL text overlay on erased PDF backgrounds produces garbage. Wrong fonts, wrong sizing, wrong positioning, poor erasing artifacts, wrong logo. It's NOT design work — it's bad automation masquerading as design.

**How to apply:**
- For ANY visual output (menus, social posts, ads, PDFs): use reference-based AI generation
- PIL = resize + save ONLY. Never for creative/design work.
- Always scrape references before designing anything
- Use design intelligence docs (strategy/, research/) before coding
- Audit with real eyes — the v1 menu audit missed wrong MIRRA logo, poor erasing, bad font sizes
