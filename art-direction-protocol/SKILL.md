# Art Direction Protocol — GAIA CORP-OS

## RULE: Always Present Before Generating

Before generating any significant visual asset (characters, environments, brand materials), ALWAYS:
1. Generate 3-4 style OPTIONS (quick/cheap)
2. Present to Jenn Woei with clear descriptions
3. Wait for approval/selection
4. THEN proceed with full generation

## Step 1: Moodboard Generation
- Use `gemini-3-pro-image-preview` for concept options (MANDATORY per standing order)
- Generate 4 variations of the same subject in different styles
- Keep prompts focused on STYLE DIRECTION not final art
- Label clearly: Option A, B, C, D

## Step 2: Presentation Format
When presenting options to Jenn, include:
- Option name + 1 sentence description
- Key visual elements
- "Best for:" use case
- Estimated generation cost/time

## Step 3: Lock Style Before Proceeding
After Jenn selects:
- Document chosen style in the relevant skill
- Update gaia-art-style/SKILL.md with final direction
- Generate style reference sheet (3 characters in chosen style)
- ONLY THEN generate full character set

## Change Management
When changing art direction:
- Jenn says "change" → present 4 options first
- Never assume new direction without showing options
- Keep old assets in /archive/ folder (never delete)
- Log change in memory with reason

## Applied to GAIA:
- Character art → 4 style options → Jenn picks → generate all 10
- Environment art → 3 composition options → pick → build
- Brand assets → mood concepts → approve → produce
- Dashboard UI → wireframe/mockup → approve → build

## Why This Matters
- Prevents wasted API credits on unwanted directions
- Ensures Jenn's vision is captured accurately
- Reduces iteration cycles
- Documents decision-making process
- Respects creative ownership

**Philosophy:** Show, don't assume. Let humans choose the aesthetic.
