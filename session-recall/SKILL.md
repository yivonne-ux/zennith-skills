# Session-Recall — Conversation Continuity

## Purpose

When a session is reset (token overflow, model switch, etc.), Zenni loses all conversation context. This skill:
1. **Extracts** a recap from the old session before reset
2. **Saves** it to the daily memory file so Zenni auto-reads it on next boot
3. **Preserves** pending tasks, decisions, and conversation threads

## How It Works

1. Watchdog detects session approaching token limit
2. BEFORE resetting, watchdog calls `recall.sh` to extract recap
3. Recap is saved to `~/.openclaw/workspace/memory/YYYY-MM-DD.md`
4. Zenni's standard boot sequence reads `memory/today.md` + `memory/yesterday.md`
5. Zenni resumes with full context of recent conversations

## Recap Format

```markdown
## Session Recap (auto-generated by session-recall)
**Session:** agent:main:whatsapp:direct:+60126169979
**Reset reason:** Token overflow (150K/262K)
**Time:** 2026-02-12 20:42 MYT

### Last 30 Conversation Turns
[compressed conversation history]

### Pending Tasks / Open Threads
- [extracted from assistant messages mentioning TODO, next, pending, will do]

### Key Decisions Made
- [extracted from messages with decision language]
```

## Integration with Watchdog

The watchdog (`auto-failover/scripts/watchdog.sh`) calls recall BEFORE resetting any session.
