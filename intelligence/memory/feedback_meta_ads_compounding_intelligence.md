## Rule: Compounding learning, not fresh analysis (2026-03-23)

User correction: "we keep pulling fresh data and fresh insights each time we run analysis on ads. It should be a compounding learning, improving and constant analysis on past, present and next strategy. Because if we did, you would know our sales dropped, and to consider if we should close and open anymore campaigns."

**Why:** Without compounding intelligence:
- I suggested opening NEW campaigns when sales were dropping
- I didn't know S19 (best ad at RM3.55) was paused
- I proposed RM120/day NEW test spend while SUPER WIN was burning RM190/day at RM12.41/convo
- I didn't flag that total spend collapsed from RM1,470→RM258/day after TEST campaigns paused
- Each session starts from zero, missing context that would change every recommendation

**How to apply:**
1. **START every ads session** by reading `project_mirra_meta_ads_compounding_ledger.md`
2. **Before ANY recommendation**, ask: "What did we decide last time? What happened? Did it work?"
3. **Never suggest opening new campaigns** without first checking: are existing campaigns healthy? Is ROAS recovering?
4. **Pull data WITH context** — compare to last session's numbers, not just absolute values
5. **END every ads session** by updating the ledger with: what changed, what we learned, what's next
6. **Flag sales/ROAS concerns** before discussing creative or structure changes
7. **Budget awareness** — always know total daily spend across all campaigns, compare to max (currently RM1,200/day)

**The operating principle:** Every session should make the NEXT session smarter. If I can't explain what changed since last time and why, I haven't done my job.
