# Universal Design System — Grid-Aesthetic NANO Production

Proven in Bloom & Bare v16 (18/18 success, 0 failures). Apply to any brand by swapping brand DNA.

---

## 1. ARCHITECTURE

**ALL-NANO**: Every design rendered by `fal-ai/nano-banana-pro/edit`. ZERO PIL text rendering.
**PIL only for**: post-processing (force_size → editorial_grade → paper_texture → place_logo → add_grain → sharpen)
**Finals only**: Never save `_raw.png`. One output per design.

---

## 2. INSTAGRAM GRID STRATEGY

### Row Rhythm Pattern
Every row of 3 posts follows: **BOLD BG | PHOTO | CLEAN TYPE**

```
Position 1: Character/illustration on bold colored background
Position 2: Photo-forward (overlay, hybrid, mascot integration)
Position 3: Type-dominant (quote, educational, interactive)
```

This creates a visual heartbeat when scrolling the profile — never 3 dark or 3 text-heavy posts stacked.

### Grid Thumbnail Safety
- IG crops 4:5 → 3:4 for profile grid (removes ~32px from each side)
- Keep all critical elements within center 1015px of 1080px canvas
- Mascot faces, key illustrations, headlines must be CENTER-positioned
- Test: would this still read if you cropped 32px from left and right?

### Color Flow
- Rotate brand colors through Position 1 across rows (creates rainbow waterfall effect)
- Position 2 (photos) creates warm consistent temperature
- Position 3 (cream/light bg) provides breathing room
- 60-30-10 rule: 60% dominant bg, 30% secondary, 10% accent

### The Squint Test
Squint at the grid until text is unreadable. The color blocks, weight distribution, and rhythm should look harmonious. If you see random patches of wildly different colors/densities, redesign.

## 3. PROMPT DNA BLOCKS (universal — swap brand values)

### Required in EVERY prompt:

**ANTI_RENDER** — Prevents NANO from rendering hex codes, font names, pixel values as visible text. Only render copy in quotes.

**LOGO_SAFE** — Top 12-15% of canvas MUST be completely clear. All headlines start below safe zone. All illustrations stay below safe zone.

**GRID_SAFE** — Most important visual element centered horizontally. Nothing critical in outer 32px left/right.

**FONT_DNA** — Describe the brand's headline + body type in sensory terms (chunky, rounded, sculptural, geometric, warm). Reference real-world analogues. Specify what it is NOT.

**TYPE_MASTER** — Headlines MASSIVE (40-60% width). Tight line spacing on display. Comfortable body spacing. ALL CAPS = wide letter-spacing. Max 2 typefaces, 3 sizes.

**LAYOUT_MASTER** — Portrait 4:5 (1080x1350). Generous margins (80px min). 40-60% breathing space. Max 4-5 elements. ONE focal point. 3-level hierarchy in 3 seconds.

**COLOR_MASTER** — Max 2-3 colors + background + text color. 60-30-10 distribution. Colors feel [brand adjective — e.g., chalky, Scandinavian, warm, muted].

### Category-specific DNA blocks:

**MASCOT_HERO_DNA** — Character fills 60-80%. Solid flat bg. ZERO patterns. ONE headline. Maximum negative space. $50K agency quality.

**CHARACTER_DNA** — Characters can be ANY type (mascots, abstract shapes, imaginary creatures, simple character illustrations). What MUST be consistent: palette, simple facial features, bold shapes, flat colors with texture, premium quality. NOT realistic humans. NOT clip-art.

**COMIC_DNA** — Clean panels, thin dividers, cream frame. Captions above/below panels (not cluttered speech bubbles). Risograph texture. Simple characters. Editorial magazine quality.

**MEME_DNA** — Internet-native but agency-level execution. Brand illustration, not stock. Clean intentional typography (NOT Impact font). Joke lands in under 2 seconds.

**ILLUSTRATED_DNA** — Editorial illustration quality. Risograph/woodblock texture. Limited palette. Bold geometric shapes. Illustration = hero (60-70% canvas). ONE headline integrated naturally.

**PHOTO_DNA** — SACRED photograph rule: keep EXACTLY as-is. Full bleed background. Overlay band (cream/brand color, ~85% opacity, organic feathered edge) for text zone. Photo fills top 65-70% unobstructed.

## 5. CHARACTER VARIETY SYSTEM

**Brand constant, design variable.**

Characters can be:
- Brand mascots (geometric shapes with faces)
- Abstract creatures (blobs, squiggles, teardrops)
- Imaginary characters (new shapes inspired by mascot style)
- Simple character illustrations (NOT realistic humans)

What MUST be consistent across ALL:
1. Same palette (brand colors only)
2. Same face style (simple googly eyes, curved smile)
3. Same quality level (premium, not clip-art)
4. Same warmth/personality
5. They all feel like they inhabit the SAME brand world

Reference standard: Headspace, LINE Friends, Jellycat, Duolingo

## 6. ANTI-PATTERNS (things that ALWAYS fail)

1. **AI-generated humans** — Always obviously AI. Use abstract characters instead.
2. **Realistic illustrated children** — Uncanny valley. Use simple shapes with faces.
3. **Watercolor/editorial illustration** — Can drift from brand. Stay bold and flat.
4. **Grid/alignment charts without art mastery** — Look generic AI. Each cell must be individually art-directed.
5. **Text in outer 32px** — Gets cropped in grid thumbnail.
6. **Headlines starting at pixel 0** — Logo safe zone violation.
7. **Hex codes, font names in output** — ANTI_RENDER block prevents this.
8. **Multi-pass NANO** — Compounds errors. Single pass only.
9. **NANO rendering logos** — Always composite real PNG in post-processing.
10. **Saving _raw.png** — Waste of tokens/storage. Finals only.
11. **3+ dark posts in a row** — Breaks grid silhouette. Alternate light/dark.
12. **No breathing space** — Designs need 40-60% whitespace to feel premium.
13. **Too many text elements** — Max 4-5 elements per LAYOUT_MASTER. Adding more degrades NANO text quality AND kills breathing space. Consolidate info into ONE compact block instead of separate layers.
14. **Percentage/pixel layout plans in prompts** — NANO interprets these literally and renders artifacts. Use natural language positioning only.
15. **Lazy illustration prompts** — Thin/generic character descriptions = generic AI output = wasted tokens. Every character needs specific expression, pose, accessories, personality. Every scene needs rich object detail, storytelling props, visual variety.

## 7. INFO-DENSE CONTENT (workshops, events, promos)

When a design has lots of info (dates, times, prices, locations, CTAs):

### The Rule: Consolidate, Don't Fragment
- **MAX 4-5 elements** — badge + headline + ONE info block + illustration/character
- Info block = ONE text group with multiple lines (not separate elements)
- NANO quality degrades with every additional text element — 4 is premium, 8 is flyer

### Typography for Info Blocks
- **ALL CAPS with VERY WIDE letter-spacing** — the #1 trick for luxury body copy
- Reference: Aesop packaging, Kinfolk magazine, Apple product pages
- Medium weight (not light, not bold)
- Generous line spacing between info lines
- Dot/pipe separators: "25-26 MARCH · 10AM-4PM · AGES 5+"

### What to include vs. omit
- INCLUDE: dates, time, price, location, age, CTA ("Limited Seats")
- OMIT from design: registration links, full descriptions, activity lists (put in caption)
- Exception: if user explicitly wants ALL info, consolidate into compact block

### Proven pattern:
```
Badge: "[EVENT/EDITION TAG]"
Headline: "[WORKSHOP NAME]" (massive, 2-3 lines)
Info block (one group, ALL CAPS, wide tracking):
  "[DATE] · [TIME] · [AGE/AUDIENCE]"
  "[PRICE TIERS]"
  "[VENUE] · [CTA]"
Character/illustration (50-55% canvas)
```

## 8. POST-PROCESSING PIPELINE

```
force_size(1080x1350)
→ editorial_grade(desat=0.10, contrast=1.08)
→ paper_texture(strength=0.025)
→ place_logo(autocrop_alpha → resize → composite)
→ add_grain(strength=4.0)  ← ALWAYS LAST before sharpen
→ sharpen_output(unsharp_mask r=1.0, pct=85, thresh=4)
```

**Logo rules:**
- Auto-crop alpha bbox before resizing (PNGs have massive transparent padding)
- 120px exclusion zone from any text/illustration
- Smart variant selection: BW logo on dark backgrounds, color on light
- Consistent position across all posts in a batch (or smart zone analysis per design)

## 8. APPLYING TO A NEW BRAND

### Step 1: Define Brand DNA
```python
BRAND = {
    "name": "Brand Name",
    "palette": {"primary": "#HEX", "secondary": "#HEX", ...},
    "bg_color": "#HEX",
    "text_color": "#HEX",
    "headline_font_desc": "describe the feel, not the name",
    "body_font_desc": "describe the feel",
    "tone": ["adjective1", "adjective2", "adjective3"],
    "characters": {"name": path, ...},  # or None
    "logos": {"variant": path, ...},
    "photos": {"name": path, ...},  # real brand photos
    "references": Path("scraped/by-category/"),
}
```

### Step 2: Generate DNA blocks
Swap all hex codes, font descriptions, tone adjectives, character descriptions into the universal DNA blocks.

### Step 3: Plan grid
Design in rows of 3. Assign each design to BOLD | PHOTO | CLEAN position. Rotate brand colors through BOLD positions.

### Step 4: Run batch
18 designs = 6 rows = ~2 weeks of content at 3 posts/week.

## 9. ENGAGEMENT OPTIMIZATION

**Highest save formats:** Educational carousels, checklists, how-to guides, fact cards (V16-09 pattern)
**Highest share formats:** Relatable memes (V16-10), tag-a-friend, interactive polls (V16-06)
**Highest comment formats:** This-or-that, quizzes (V16-06), bingo (V16-15), caption-this

Plan content mix: 30% saveable educational, 40% shareable meme/interactive, 30% brand world (character hero, illustrated scenes)

## 10. QUALITY SIGNALS (premium vs cheap)

| Premium | Cheap |
|---------|-------|
| 1-2 fonts, consistent hierarchy | 3+ fonts, no system |
| 40-60% whitespace | Content fills 90%+ |
| 3-5 colors from defined palette | 8+ random colors |
| Custom illustration, consistent style | Mixed stock/clipart |
| Consistent photo treatment | Random editing |
| Visible grid rhythm | Random sequencing |
| Template system (recognizable post types) | Every post redesigned from scratch |
| Subtle consistent logo | Logo too large, inconsistent |
| Concise punchy copy | Verbose, crammed text |
| Unified texture/grain treatment | Random mix of filters |
