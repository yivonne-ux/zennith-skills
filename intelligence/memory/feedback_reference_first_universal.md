## REFERENCE-FIRST PRINCIPLE — UNIVERSAL

### The Rule
NEVER generate ANY visual content from imagination alone. ALWAYS start from a reference.

### Why
- Generating from scratch = NANO defaults to its training data average = generic AI look
- Reference image as Image 1 = NANO has a specific target to match = branded output
- This is why W1-04-S1 (used itself as ref) produced better results than prompting from scratch
- This is why v5 illustration (sweater girl ref) works better than describing "warm illustration"

### How to Apply

**For food hero posts:**
- Image 1: Mood reference photo (color/lighting anchor)
- Image 2: Real food cutout (dish anchor)
- NANO edits TOWARD the reference, not from scratch

**For illustration posts:**
- Image 1: Character style reference (face/proportion anchor)
- Image 2: Second character ref OR style ref
- NANO adapts the reference style, not inventing one

**For carousel slides:**
- Use a PROVEN slide as Image 1 reference for new slides in same series
- E.g., W1-04-S1 as ref for illustration-led hook slides
- E.g., a good food hero slide as ref for other food slides

**For new post types:**
- Find a reference from Pinterest, competitors, or your own past work FIRST
- Upload it as Image 1
- Prompt NANO to adapt it, not create from nothing

### The Hierarchy
1. Best: Your own past approved output as reference (proven to work)
2. Good: Curated Pinterest/competitor reference (proven in market)
3. Acceptable: Mood/style reference from ref library
4. NEVER: No reference, prompt-only generation

### Compound Learning
Every approved output becomes a reference for future generations. The library of "what works" grows over time. This is how the engine gets better — each generation feeds the next.
